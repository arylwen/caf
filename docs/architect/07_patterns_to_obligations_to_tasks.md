# Patterns → obligations → tasks

This doc explains the “middle” of CAF: how decisions become work.

## Promotion (patterns → obligations)

- Adopted patterns imply **obligations**.
- Obligations are explicit objects (not freeform text).

Primary artifact:

- `design/playbook/pattern_obligations_v1.yaml`

Derived index:

- `design/playbook/pattern_obligations_index_v1.tsv` (generated by `tools/caf/gen_obligations_index.mjs`)

See also:

- `technical_notes/TN-012_promotions_and_obligations_v1.md`

## Planning (obligations → tasks)

- The planner produces `task_graph_v1.yaml`.
- Gates enforce:
  - required capability coverage
  - trace anchors for promoted obligations
  - contract coherence

Primary artifacts:

- `design/playbook/task_graph_v1.yaml`
- `design/playbook/task_plan_v1.md` (derived view)

Derived index:

- `design/playbook/task_graph_index_v1.tsv` (generated by `tools/caf/gen_task_graph_index.mjs`)

## Where “how big is the work” comes from

CAF sizing uses:

- counts + grouping on obligation/task indexes
- graph topology (waves)
- capability distribution (optionally mapped to workers)

See: [`03_work_visibility_sizing.md`](03_work_visibility_sizing.md)
