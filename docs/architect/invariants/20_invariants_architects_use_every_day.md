# Twenty invariants architects use every day

Architects and developers carry hundreds of invariants every day. CAF helps make that discipline explicit.

These twenty are a good place to start because they sit close to the questions people care about first: tenancy, context, trust, security, auditability, resilience, AI safety, and boundary discipline.

## The short list

1. **Cross-tenant exposure through shared runtime surfaces is forbidden unless it is explicitly scoped and justified.** A cache, log stream, search index, or shared worker can quietly become the place where tenant isolation fails first. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-02-a4) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
2. **Required runtime configuration is validated centrally and fails closed at startup.** Teams should not discover missing keys and broken secrets one request at a time in production. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-05-a2) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
3. **The tenancy model is explicit.** Resources belong to a tenant on purpose. Workspaces, cohorts, and convenience grouping are not substitutes for tenant authority boundaries. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-02-a1) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
4. **Tenant context is bound once and propagated end to end.** Architects do not want tenant context guessed differently by each layer. They want it established at a trusted boundary and then carried forward consistently. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-02-a2) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
5. **Missing or ambiguous tenant scope fails closed.** When required scope is missing, the safe answer is not “best effort.” The safe answer is “stop.” [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-02-a3) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
6. **Trust validation happens early and consistently.** Identity claims, tokens, and other trust inputs should not be validated in five different half-compatible ways. Architects need a governed trust posture, not a collection of local opinions. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-03-a2) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
7. **Significant actions remain attributable.** If the team cannot reconstruct who did what, in which tenant, under which authority, then policy and audit become weaker than they look. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-03-a3) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
8. **Agents are governed principals too.** If the system includes agents, they cannot live outside the identity model. They need governed identity, bounded authority, and explicit context just like human and service actors do. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-03-a4) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
9. **Protected operations go through explicit policy enforcement points.** A protected operation should not have one official path and three side doors. Architects need to know where policy is enforced and where bypass is impossible. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-04-a1) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
10. **Application and Data Plane code do not bypass Control Plane policy authority.** Everyone “knows” policy belongs somewhere. But unless the workflow keeps that authority split visible, downstream code starts redefining policy locally. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-04-a3) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
11. **Persistence and integration sit behind explicit ports.** Architects need domain code insulated from storage and integration details. That is about keeping core meaning stable when adapters, vendors, or runtime shapes change. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-05-a1) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
12. **Resource operations enter through explicit service facades.** If resource behavior can be reached through ad hoc paths with different validation and semantics, the architecture becomes hard to reason about. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-05-a3) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
13. **Telemetry carries correlation and relevant tenant or user identifiers.** Observability is much less useful when traces cannot be tied back to the architectural context that produced them. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-06-a1) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
14. **Audit events are immutable and scoped.** Audit trails are only helpful when they are durable, attributable, and scoped correctly. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-06-a2) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
15. **Tenant, principal, and correlation context propagate across the UI boundary.** UI responses and actions need the same trust, scope, and traceability discipline as the services behind them. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-07-a2) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
16. **Safety Gates are first-class and binding.** AI flows that bypass safety-gate ownership and outcome discipline are not a clever shortcut; they are an architectural violation. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-12-a2) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
17. **Planes keep clear responsibilities and zero-trust boundaries.** Control Plane, Application Plane, and Data Plane should not blur into one another just because a codebase makes that convenient. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-11-a1) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
18. **Runtime resilience is intentional, not accidental.** Retry-safe side effects, deadlines, bounded retries, degradation posture, and quarantine handling all need to be architectural choices. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-14-a3) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
19. **AI policy, allowed models, and tool permissions are explicitly governed.** The moment AI enters the system, architects inherit a new class of invariants: which models are allowed, which tools are allowed, and which outputs are trusted. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-12-a1) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)
20. **Secrets stay externalized.** If secrets leak into code, images, prompts, or ad hoc runtime conventions, the system stops being safely operable. [Catalog row](target_system_invariant_catalog_v1.md#inv-tsys-shape-fam-15-a1) · [Machine-readable](../../../architecture_library/16_contura_target_system_invariant_catalog_v1.yaml)

## Why these twenty work as an entry point

They are not the whole invariant catalog. They are the rows that most quickly make the point that architecture is really a managed set of truths: some always on, some shaped by pins, and some sharpened only when a specific option branch is adopted.
